# 07. README Rewrite Plan

## Objective

Turn the README from a vision statement into a reproducible entry point for users and contributors.

## Current Findings

- The current README explains the concept well.
- It does not yet provide enough setup and verification detail for an external user.

## Target README Structure

1. What this project is
2. Current maturity and intended use
3. Quick start
4. Installation
5. Running backend and frontend
6. Sample input and sample output
7. API usage examples
8. Generated artifact examples
9. Directory layout
10. Security and data-handling notes
11. Evaluation and limitations
12. Non-goals

## Action Items

1. Add a concise project summary at the top.
2. Add exact install and run commands.
3. Add one end-to-end example using bundled sample data.
4. Show one API request and one API response.
5. Explain where local artifacts and user data are stored.
6. Link to evaluation docs and security policy.
7. Add limitations and non-goals.

## Acceptance Criteria

- A new user can set up and run the project from README alone.
- A reviewer can quickly understand scope, outputs, and current limits.
